# 咖啡荒谬人格研究所 V2.2

这是一个借鉴“娱乐化人格测试传播机制”的咖啡人格网站。它不复制任何具体测试题或视觉资产，而是把有梗结果、隐藏人格、可分享海报与真实咖啡消费画像结合起来。

## 直接运行

解压后双击 `index.html` 即可。

为避免部分浏览器限制本地文件的分享和剪贴板功能，也可以在目录内启动一个简单静态服务器：

```bash
python -m http.server 8080
```

然后访问 `http://localhost:8080`。

## V2.3 已测试人数

首页顶部栏中间新增“已测试人数”。

配置位于 `app.js` 顶部：

```javascript
const TEST_COUNT_CONFIG = {
  initialCount: 1286,
  endpoint: ""
};
```

- 只展示固定人数：修改 `initialCount`。
- GitHub Pages 本身无法保存全站真实人数。
- 需要真实人数时，将 `endpoint` 设置为统计接口地址。
- 接口应支持：
  - `GET` 返回 `{ "count": 1286 }`
  - `POST` 返回递增后的 `{ "count": 1287 }`
- 同一浏览器只会在首次完成测试时请求递增一次。

移动端结果海报改为多级兼容：

1. 支持 Web Share 文件分享的手机浏览器：

说明：GitHub Pages 是静态托管，不能直接完成微信公众号 JS-SDK 的服务端签名。
如需在微信网页内直接配置“发送给朋友/分享到朋友圈”的标题、图片和链接，
需要认证公众号、JS 安全域名及服务端签名接口。


## V2.5 海报二维码

生成的结果海报现在会自动附带网站二维码。

实现方式：

- 海报生成时，使用当前网站地址（去掉 `#` 片段）作为二维码内容
- 用户识别二维码后即可回到测试网站
- 若二维码库加载失败，海报仍会正常生成，只是二维码位置会显示失败提示

说明：

- 当前二维码依赖页面底部加载的 `qrcodejs` 库
- 链接内容默认使用当前页面地址
- 如果以后切换正式域名，不需要改代码，二维码会自动使用新域名

## V2.5.3 真正连续循环

当前滚动轨道包含三组完全相同的人格名称。

动画由移动 `-50%` 改为移动 `-33.333333%`，即每次只移动一整组的宽度。动画重置时，第二组所在的位置与第一组完全一致，因此不会出现跳动或断层。

同时增加：

- `flex-wrap: nowrap`
- `backface-visibility: hidden`
- `translate3d` GPU 加速
- 动画速度调整为 30 秒，滚动更平稳
